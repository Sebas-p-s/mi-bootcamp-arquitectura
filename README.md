# [Sistema Rechum] - Proyecto Bootcamp

## 1️⃣ ¿Qué Problema Resuelve?

Una necesidad de una falta de orden en el area de recursos humanos, además ayudaría al personal de recursos humanos con todas las tareas que deben cumplir.

También como un apartado importante seria el que facilitaría el tener todo separado con sus relacionados, ejemplo: Juan un trabajador tendría todos sus archivos e historial a un solo click, y con su propio espacio para que no se confunda con un trabajador

Lo ideal es que solo lo llegue a usar el personal de RR.HH para que sus tareas sean más fáciles, pero pensando en quienes lo van a pagar seria ideal que los jefes de las empresas también tengan acceso para tener control.

Es importante porque seria un sistema el cual haría  más eficiente y ordenado lo que hacen en RR.HH esto ayudaría a la empresa y esta area.

## 4️⃣ Decisiones Iniciales

### Metodología de Desarrollo

**Elegí**: Kanban

**¿Por qué?**: Siento que la metodología Kanban tiene esa garantía para mi proyecto de no depender de tener demasiadas personas en el equipo de trabajo, además de no depender de unos roles asignados sino que dependemos de que nos repartamos bien las tareas que se están haciendo, las que se harán y las que ya terminemos. En palabras fáciles no dependemos tanta cantidad de personas trabajando sino de las tareas que escojamos hacer en un orden además de un retroalimentación en nuestros errores.

### Arquitectura Inicial

**Elegí**: Monolito 

**¿Por qué?**: Facilitara en un futuro el tener tanto código y roles diferentes de nuestro software, además es mucho mejor que tener todo revuelto como en un monolito. En palabras sencillas evitaremos tener un revuelto que pierde sentido entre más roles haya y tendremos mejor un sistema ordenado y fácil de escalar.

### Tecnologías

- **Backend**: Node.js (porque ya lo conocemos)
- **Base de datos**: PostgreSQL (porque es gratis y potente)
- **Frontend**: React (porque tiene mucha documentación)
